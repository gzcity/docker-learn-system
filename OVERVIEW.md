## 核心设计一瞥

### 一句话概括

> **这不是一个"带 Docker 学习功能的聊天机器人"。这是一个由知识图谱驱动、多智能体协作、上下文持续流转的 Docker 源码学习操作系统。**

### 三个关键洞察

1. **知识图谱是心脏，不是数据库**  
   不是"存数据的"，而是"发现关系的"。概念之间的 `:PREREQUISITE`、`:CALLS`、`:IMPLEMENTS` 关系，决定了学习路径、测验范围和可视化结构。

2. **上下文是血液，不是缓存**  
   用户从"对话"切换到"测验"再切换到"可视化"时，当前概念、掌握度、人格偏好、未完成题目——这些不是"页面间传递的参数"，而是 Agent 内部状态的**自然延续**。

3. **Agent 是神经元，不是页面**  
   用户不说"打开题库"或"去知识库页面"，而是说"考考我关于 cgroups 的内容"——Agent 自主决定调用题库、检索知识库、更新掌握度。用户感觉不到"功能切换"。

### 最简启动路径

```
P0 (1天) → 可对话，可检索知识图谱
P1 (2天) → 可看书，可记笔记
P2 (2天) → 可测验，可自动评分
P3 (2天) → 掌握度追踪，间隔重复练习
P4 (3天) → 源码分析，可视化生成
P5 (1天) → 跨会话记忆，上下文恢复
P6 (1天) → 人格切换，自适应路径
P7 (2天) → 学习闭环，体验打磨
─────────────────────────────
总计：约 14 天 → 完整系统
```
