> 什么是容器运行时？

🧠 容器运行时: 负责管理容器完整生命周期的底层软件组件。
核心组成：runc → 创建容器进程，containerd → 管理生命周期，shim → 桥梁
源码路径：docker/daemon/ → containerd API → runc create/start

## 🎯 为什么需要这个？

学习 Docker 源码有三个痛点：

1. **Docker 204 万行 Go 代码** — 从哪开始？
2. **现有工具要么是文档（被动），要么是教程（线性）** — 都不适应你的水平
3. **学了就忘** — 没有间隔重复，大脑会遗忘

**这个系统三个都解决了。** 就像有个私人导师：

| 能力 | 说明 |
|------|------|
| 📖 了解代码库 | 概念映射到实际源码文件 |
| 🧠 记住你学了什么 | 自适应难度和节奏 |
| 🔄 在正确时间复习 | SM-2 间隔重复算法 |
| 🎭 用你的风格教学 | 8 种人格预设 |
| 🔍 深入挖掘 | 生成带架构图的研究报告 |

## 🎭 8 种教学人格

| 人格 | 风格 | 示例 |
|------|------|------|
| **苏格拉底** | 反问引导，不直接给答案 | "你觉得这里为什么用接口？" |
| **教授** | 结构化、系统化、严谨 | "从三个层面理解容器运行时..." |
| **实践者** | 代码驱动、动手导向 | "打开 runc/create.go 第 42 行..." |
| **说书人** | 类比驱动、故事化 | "容器是公寓，cgroups 是物业..." |
| **教练** | 目标导向、激励驱动 | "今天目标拆成 3 步..." |
| **调试者** | 问题导向、逆向思考 | "如果这里 panic 了，调用栈是？" |
| **极简者** | 最简回答、直奔重点 | "runc create 创建，run = create+start" |
| **唱反调** | 挑战观点、引发思考 | "你真的确定 overlay2 更好吗？" |

切换方式：
```text
> 用教练风格
> 苏格拉底模式
> 这次用极简者的方式
```

## 🧠 间隔重复 (SM-2)

```
掌握度记录 "容器运行时":
  掌握度:     0.65 (65%)
  上次练习:   2 天前
  练习次数:   5 (3/5 正确)
  下次复习:   明天 (到期!)
  遗忘曲线:   正在下降 — 现在复习！
```

**核心机制：**
- **遗忘曲线**：掌握度随时间指数衰减
- **关联传播**：一个概念提升，关联概念也提升 30%
- **到期预测**：自动识别需要复习的概念
